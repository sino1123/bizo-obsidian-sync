# Weekly Summary (2026-W15)

> 범위: KST 기준 최근 7일(2026-04-05~2026-04-11) 중 존재하는 Daily 노트 기반 요약

## 1) 이번 주 주요 도메인/토픽 하이라이트 (10줄 이내)
- **EU AI Act(GPAI) Code of Practice**가 ‘준수 추정(presumption of conformity)’ 경로를 제공: **투명성/저작권/안전·보안(리스크관리·레드팀·사고보고)** 중심.
- 2026년은 **High-risk 의무 전면 적용(2026-08-02)** 전 단계로, 공급자/도입자 준비(문서·거버넌스·증적)가 실무 쟁점화.
- **NIS2 하위 이행규정(Commission Implementing Regulation 2024/2690)**을 ENISA가 **통제→구현예시→증빙** 형태로 구체화(감사/감독 대응 직결).
- NIS2는 특히 **24h/72h 사고보고, 공급망 보안, BC/DR** 등 운영 프로세스가 ‘형식’이 아닌 **상시 운영능력(테스트/훈련/측정)**으로 평가되는 방향.
- **NIST FIPS 203(ML-KEM)** 확정으로 PQC 전환이 “알고리즘 채택”이 아닌 **crypto-agility + 공급망(SBOM/모듈검증) + 하이브리드 전환** 프로그램으로 재정의.
- **EUDI Wallet(eIDAS 2.0)** 원격 온보딩 Implementing Act로 **신원확인·디바이스 바인딩·상호운용성**이 규격 중심으로 표준화.
- **DSA 집행**에서 미성년자 보호(연령확인) 미흡에 대한 예비결론: **자기선언 1-click은 부적절**, 프라이버시 보존형 AV 요구 강화.

## 3) “1교시 후보” 5개 (주제 + 출제 포인트 1줄)
1. **EU AI Act의 GPAI 의무와 Code of Practice**
   - 규정 요구를 **MLOps 산출물(리스크레지스터/테스트증적/로그/문서)**로 변환하는 실무형 논술이 가능.
2. **NIS2 사고보고(24/72h) 체계와 운영 KPI화**
   - ‘보고 타임라인/증빙’이 곧 감사 포인트라 **프로세스+지표+훈련** 구성이 출제 친화적.
3. **PQC 전환과 ML-KEM(FIPS 203) 기반 Crypto-Agility**
   - 하이브리드 키교환, 키관리, 성능/호환성, 모듈검증까지 **전환 프로그램**으로 쓸 수 있음.
4. **EUDI Wallet 원격 온보딩(신원확인·디바이스 바인딩·상호운용성)**
   - 신원은 기술이 아니라 **책임/감사/표준 인터페이스**가 핵심이라는 관점 전개가 쉬움.
5. **DSA 관점의 연령확인(Age Verification)과 프라이버시 보존 설계**
   - UX·검증기술·감사로그를 묶어 **규제 준수 가능한 아키텍처**로 답안 구성 가능.

## 4) “2/3교시 확장거리” 3개 (아키텍처/대안/리스크/대응)
1. **규제 준수형 AI 제품/서비스 아키텍처(Compliance-by-Design MLOps)**
   - 아키텍처: 정책/법무 요구 → 통제카탈로그 → 모델카드·데이터요약·저작권정책·리스크평가·레드팀 증적 파이프라인화.
   - 리스크: 문서 누락·추적불가·사고보고 지연 → 제재/조달 실패.
   - 대응: 산출물 표준 템플릿, 로그/모니터링, 변경관리, 공급자/다운스트림 정보제공 계약(SLA) 내재화.
2. **PQC 마이그레이션 아키텍처(하이브리드 단계전환 + Crypto-Agility)**
   - 대안: (a) 하이브리드 핸드셰이크 우선, (b) 서비스군별 우선순위(외부노출/장기보관 데이터 중심), (c) 라이브러리/모듈 검증.
   - 리스크: “Kyber 구현=준수” 오인, 성능저하, 상호운용성 실패, 공급망 취약.
   - 대응: 알고리즘 추상화(Agility), 키관리/인증서/프로토콜 영향분석, SBOM/서명검증, 단계적 롤아웃.
3. **프라이버시 보존형 연령확인·디지털 신원 연계 설계(DSA × eIDAS 2.0)**
   - 아키텍처: 최소정보 공개(속성증명), 토큰/자격증명 기반 접근제어, 감사로그 분리보관, 디바이스 바인딩.
   - 리스크: 과도한 PII 수집, 다크패턴 UX, 우회/대리인증, 컴플라이언스 증빙 부재.
   - 대응: 데이터 최소화, 사용자흐름 설계(마찰/우회 방지), 제3자 검증/감사, 위협모델링/테스트.
