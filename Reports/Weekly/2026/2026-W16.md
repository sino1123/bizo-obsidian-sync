# 주간 요약 (2026-W16)

## 1) 이번 주 하이라이트(10줄 이내)
- **규제·프레임워크의 공통축은 ‘거버넌스+증빙’ 강화**: NIST CSF 2.0, DSA/DMA, Interoperable Europe Act 모두 정책 선언보다 운영 산출물과 평가체계를 요구.
- **NIST CSF 2.0은 Govern 기능 중심으로 ERM 연계가 핵심**: 참조표준 매핑, 역할·책임, KPI, 의사결정 기록이 실무 증적이 됨.
- **공공 디지털서비스는 상호운용성 평가가 설계 선행조건화**: API 공개를 넘어 표준 스키마, 코어 보캐뷸러리, QoS/약관까지 통합 설계 필요.
- **알고리즘/AI는 성능보다 책임성 통제가 시험 포인트로 부상**: XAI, 데이터 투명성, 개인정보 영향평가, 알고리즘 책임성이 함께 묶임.
- **보안은 Zero Trust와 랜섬웨어 대응이 여전히 중심축**: 클라우드 환경에서 IAM, 세그멘테이션, 데이터 보호를 아키텍처 수준에서 설명해야 함.
- **운영체제·아키텍처는 AI 내장형 플랫폼 전환이 가속**: AIOS, 엣지 컴퓨팅, AI PC, AI-Native 플랫폼이 HW/SW 동시 최적화를 요구.
- **RISC-V는 프로파일 기반 표준화가 실무 쟁점**: RVA23, ABI 안정성, 컴플라이언스 테스트가 상용화 리스크를 좌우.
- **네트워크는 5G Advanced와 AI 자동화가 결합**: 운영 자동화, 데이터 기반 최적화, 향후 6G 연계 서술이 유효.
- **Notion 키워드 측면에선 TCP 오류제어·인터미턴트 컴퓨팅·DevOps가 보완축**: 기본기형 1교시와 확장형 2/3교시를 동시에 준비하기 좋음.

## 3) 1교시 예상 TOP 5 (강화)

### 1) NIST CSF 2.0 Govern 기능 (보안/거버넌스)
- 출제 가능 포인트: 보안관리 프레임워크가 기술통제 중심에서 **경영 연계형 거버넌스와 증빙체계**로 이동 중이기 때문.
- 1교시 답안 골격: *CSF 2.0의 구조와 Govern 기능의 역할을 설명하고 조직의 책임체계·위험연계·평가증적을 정리* + 핵심 키워드 5개 + *국내 ISMS/ISO27001 연계 시사점*.
  - 핵심 키워드(5): Govern, ERM, Accountability, Informative References, KPI
  - 시사점: 프레임워크 채택 여부보다 **거버넌스 운영기록과 참조표준 매핑 산출물**이 실무 성숙도를 좌우.

### 2) Interoperable Europe Act와 상호운용성 평가 (디지털서비스/전자정부)
- 출제 가능 포인트: 공공서비스 설계에서 **기술·의미·조직 상호운용성**을 사후 연동이 아닌 사전 평가로 강제하는 흐름이 뚜렷함.
- 1교시 답안 골격: *상호운용성의 개념과 평가 절차를 설명하고 API·데이터 표준·조직 협업 관점의 설계 포인트를 정리* + 핵심 키워드 5개 + *공공 SI 적용 시사점*.
  - 핵심 키워드(5): Interoperability Assessment, Core Vocabularies, API 거버넌스, 의미론, QoS
  - 시사점: 개별 시스템 최적화보다 **표준 스키마와 평가체계 선확정**이 장기 유지보수비용을 낮춘다.

### 3) 알고리즘 책임성과 XAI (AI/컴플라이언스)
- 출제 가능 포인트: AI 활용 확산과 함께 **설명가능성, 데이터 투명성, 영향평가**가 규제 대응의 기본 문법이 되고 있기 때문.
- 1교시 답안 골격: *알고리즘 책임성의 개념과 필요성을 설명하고 XAI·PIA·감사로그 중심의 통제요소를 정리* + 핵심 키워드 5개 + *운영관리 시사점*.
  - 핵심 키워드(5): XAI, PIA, 데이터 투명성, 책임성, 글로벌 규제
  - 시사점: AI 모델의 정확도만으로는 부족하며 **설명·기록·검증 가능성**이 답안의 핵심 포인트가 된다.

### 4) Zero Trust 기반 클라우드 보안 아키텍처 (보안/클라우드)
- 출제 가능 포인트: 랜섬웨어와 하이브리드 클라우드 확산으로 **신뢰경계 재정의와 최소권한 통제**가 반복 출제 포인트가 됨.
- 1교시 답안 골격: *Zero Trust의 개념과 기존 경계형 보안과의 차이를 설명하고 IAM·세그멘테이션·지속검증 구조를 정리* + 핵심 키워드 5개 + *랜섬웨어 대응 시사점*.
  - 핵심 키워드(5): 최소권한, IAM, 마이크로세그멘테이션, 지속검증, 데이터 보호
  - 시사점: 솔루션 도입보다 **정체성 중심 정책과 자산·데이터 흐름 재설계**가 우선이다.

### 5) RISC-V 프로파일(RVA23)과 생태계 표준화 (컴퓨터아키텍처)
- 출제 가능 포인트: 오픈 ISA 확산 국면에서 **프로파일 기반 호환성 확보와 상용화 리스크 관리**가 아키텍처형 문제로 적합함.
- 1교시 답안 골격: *RISC-V 구조와 프로파일 개념을 설명하고 ABI 안정성·배포판 기준선·컴플라이언스 관점을 정리* + 핵심 키워드 5개 + *제품화 시사점*.
  - 핵심 키워드(5): RISC-V, RVA23, Profile, ABI 안정성, Compliance Test
  - 시사점: ISA 확장 자유도는 장점이자 위험이므로 **표준 프로파일 준수와 검증 자동화**가 필수다.

## 4) 2/3교시 확장거리 3개 (아키텍처/대안/리스크/대응)
1. **NIST CSF 2.0 기반 보안 거버넌스 재설계**: Govern-Identify-Protect-Detect-Respond-Recover 연결, ERM/RACI/KPI 체계, ISMS·ISO27001·NIST 800-53 매핑, 증적 자동수집 구조.
2. **공공 디지털서비스 상호운용 아키텍처**: 표준 API 게이트웨이, 메타데이터/코어 보캐뷸러리, 조직 간 데이터 거버넌스, 상호운용성 평가 프로세스와 품질관리 방안.
3. **AI/클라우드 통합 리스크 대응**: 알고리즘 책임성(XAI·영향평가) + Zero Trust 클라우드 보안(IAM·세그멘테이션) + DevSecOps(IaC·CI/CD 보안검증)를 통합한 운영모델.
