# 2026-04-23 IT 데일리 리포트

[TurboQuant 기반 초고압축 벡터/LLM 메모리 최적화] (알고리즘)

출처 URL: https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/

내용 요약: Google Research가 2026-03-24 공개한 TurboQuant는 PolarQuant, QJL을 결합해 KV cache와 벡터 검색 메모리를 크게 줄이는 압축 알고리즘입니다.
최대 6배 수준의 메모리 절감, H100 기준 attention 연산 8배 가속 사례를 제시했습니다.
정확도 저하를 거의 억제해, 대규모 AI 추론 인프라의 병목을 알고리즘적으로 완화하는 흐름이 뚜렷합니다.

핵심 키워드 5개: TurboQuant, PolarQuant, QJL, KV cache, 벡터검색압축

기술사적 관점(Insight): 알고리즘 최적화는 하드웨어 증설보다 빠르게 TCO를 낮출 수 있으므로, 성능평가 시 정확도-지연시간-메모리 3축 검증 체계를 같이 설계해야 합니다.

[Windows 커널 드라이버 신뢰정책 전환] (OS)

출처 URL: https://techcommunity.microsoft.com/blog/windows-itpro-blog/advancing-windows-driver-security-removing-trust-for-the-cross-signed-driver-pro/4504818

내용 요약: Microsoft는 2026년 4월부터 구형 cross-signed 드라이버의 기본 신뢰를 제거하고, WHCP 서명 중심으로 Windows 커널 정책을 전환하고 있습니다.
초기에는 evaluation mode와 allow list를 병행해 호환성 충격을 줄이되, 장기적으로는 커널 모드 공격면을 줄이는 방향입니다.
내부 전용 드라이버는 WDAC(Application Control for Business) 기반 예외 관리가 사실상 표준 대응이 되고 있습니다.

핵심 키워드 5개: Windows kernel, cross-signed driver, WHCP, WDAC, evaluation mode

기술사적 관점(Insight): 운영체제 보안은 기능 추가보다 신뢰루트 정비가 핵심이며, 레거시 드라이버 자산은 인증체계 전환 계획 없으면 장애와 보안 리스크가 동시에 커집니다.

[OWASP Agentic Applications Top 10 2026] (SW공학)

출처 URL: https://genai.owasp.org/resource/owasp-top-10-for-agentic-applications-for-2026/

내용 요약: OWASP는 2026년 Agentic Applications Top 10을 통해 목표 하이재킹, 도구 오남용, 메모리/컨텍스트 오염, 에이전트 공급망 취약점 등을 핵심 리스크로 제시했습니다.
AI 코딩 에이전트와 자율형 워크플로가 늘면서 기존 웹 취약점 점검만으로는 충분하지 않다는 점이 공식화됐습니다.
설계 단계부터 권한분리, 승인 게이트, 도구 호출 통제, 메모리 검증을 포함한 SSDLC 확장이 요구됩니다.

핵심 키워드 5개: Agentic AI, OWASP Top 10, Goal Hijack, Tool Misuse, Context Poisoning

기술사적 관점(Insight): SW공학 관점에서 에이전트 보안은 구현 이슈가 아니라 요구사항·아키텍처·검증 프로세스 전체를 다시 정의하는 품질 속성입니다.
