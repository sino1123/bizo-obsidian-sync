# 데일리 글로벌 IT 동향 리포트 (2026-04-11, KST)

[EU AI Act GPAI Code of Practice/가이던스 진행상황] (AI)

출처 URL: https://digital-strategy.ec.europa.eu/en/policies/contents-code-gpai

내용 요약: EU AI Act의 General-Purpose AI(GPAI) 모델 의무 준수를 돕기 위한 Code of Practice가 운영되며, 투명성·저작권·안전/보안(리스크관리, 레드팀, 사고보고 등) 항목을 중심으로 ‘준수 추정(presumption of conformity)’ 경로를 제공. 2026년은 고위험(High-risk) 의무 전면 적용(2026-08-02)을 앞두고 공급자/도입자 준비가 실무 쟁점화되는 시기.

핵심 키워드 5개: EU AI Act, GPAI, Code of Practice, presumption of conformity, red teaming

기술사적 관점(Insight): 규제 준수는 ‘모델 성능’보다 문서화/거버넌스/사고대응 체계를 먼저 요구하므로, 요구사항을 SDLC/MLops 산출물(리스크레지스터·테스트증적·로그·SLA)로 변환하는 능력이 합격 포인트.


[NIST FIPS 203(ML-KEM) 기반 PQC 전환 가속] (보안)

출처 URL: https://csrc.nist.gov/pubs/fips/203/final

내용 요약: NIST가 ML-KEM(FIPS 203)을 최종 표준으로 확정하면서(구 Kyber 대비 규격 변경 존재), 조직은 ‘이전 Kyber 구현=준수’로 오인하지 않도록 검증/교체가 필요. 2026년은 레거시 공개키(RSA/ECC) 단계적 폐지 로드맵(2030/2035)과 맞물려 하이브리드 핸드셰이크, 라이브러리/모듈 검증, 제품 카테고리별 우선순위화가 실무 과제로 부상.

핵심 키워드 5개: Post-Quantum Cryptography, FIPS 203, ML-KEM, hybrid key exchange, crypto-agility

기술사적 관점(Insight): PQC는 ‘알고리즘 채택’이 아니라 공급망(SBOM/모듈인증)과 크립토-애질리티(키관리·프로토콜·성능)까지 포함한 전환 프로그램으로 설계해야 리스크가 줄어든다.


[EU EUDI Wallet 원격 온보딩 Implementing Act 공개(신원확인·디바이스 바인딩 표준화)] (디지털서비스)

출처 URL: https://www.biometricupdate.com/202604/eu-publishes-implementing-act-for-remote-eudi-wallet-onboarding

내용 요약: EU가 EUDI Wallet 사용자 원격 온보딩 관련 Implementing Act를 통해 신원검증 및 개인식별데이터(PID)와 지갑/디바이스의 신뢰 결합을 위한 참조 표준·스펙을 정의. 지갑의 상호운용성과 고신뢰(고보안) 인증 체계를 ‘실행규칙’ 수준으로 구체화하면서, eKYC/생체인증/원격본인확인 설계가 규격 중심으로 재편되는 흐름.

핵심 키워드 5개: eIDAS 2.0, EUDI Wallet, Implementing Act, remote onboarding, interoperability

기술사적 관점(Insight): 디지털 신원은 기술보다 제도(인증/책임/감사)와 표준 상호운용성이 승부처라서, 공공서비스 설계 시 ‘규정 준수 가능한 아키텍처(온보딩·증명서·서명·감사로그)’를 먼저 잡아야 한다.
